## Findings surfaced but not planned (this run)

High-value, worth their own future plans:

- **[SEC] Task attachments served with no auth** — `/api/uploads` static mount at
  `backend/app.js:214,217` is registered before `requireAuth`; any unauthenticated
  party with a (guessable) URL downloads any tenant's files. HIGH impact.
- **[BUG] Backup restore breaks on any attachment** — `services/backupService.js:469-476`
  writes `file_name/file_url/file_type` but the model requires
  `original_filename/stored_filename/mime_type/file_path` (allowNull:false) →
  throws, rolls back the whole import. Core restore broken for anyone who
  attached a file. Small fix, HIGH leverage.
- **[BUG] Backup FK remap uses stale numeric IDs** — `backupService.js:341-347,401-407,497-521`
  relinks parent/project/area by old numeric `id` (not the UID map) and without
  `user_id` scoping → cross-instance restores mis-link or bind to another user's
  rows.
- **[SEC] SSRF in URL metadata fetch** — `modules/url/service.js:229-289,388-399`
  fetches user URLs and follows redirects with no private-IP guard (authed-only).
- **[SEC] OIDC links to existing account on unverified email** —
  `modules/oidc/provisioningService.js:73-90` matches an existing local user by
  `claims.email` with no `email_verified` check → account-takeover risk depending
  on IdP trust.
- **[SEC] No session regeneration on login** (fixation) — `auth/service.js:207`,
  `oidc/controller.js`.
- **[BUG] OIDC auto-admin no-op** — `provisioningService.js:94-100` runs
  `Role.update` on a just-created user with no Role row → 0 rows.
- **[TECH-DEBT] `tasks` god-module** — `modules/tasks/routes.js` 1056 lines, PATCH
  handler ~355 lines inline, raw `PRAGMA foreign_keys` toggling in the HTTP
  handler; only module without service/controller. Needs characterization tests
  first (dependency on a future test plan).
- **[TECH-DEBT] area/tag sharing stubbed but offered by FE** —
  `permissionsCalculators.js:185-191` no-ops; `frontend/utils/sharesService.ts:7`
  types `area|tag` → backend returns Forbidden. Prune or implement.
- **[DX] tsconfig `strict: false`** — `tsconfig.json:8`; enable incrementally.
